# 数据同步架构

## 核心表

- `platform_connections`
- `sync_jobs`
- `sync_runs`
- `feishu_exports`
- `audit_logs`

## 同步流程

1. 平台方或客户管理员配置连接。
2. 创建同步任务。
3. 定时任务拉取或接收 webhook。
4. 数据清洗并写入主数据库。
5. 记录 `sync_runs`。
6. 生成 AI 输出或任务。
7. 必要数据同步到飞书。

## 失败处理

- 保存错误原因。
- 支持人工重试。
- 飞书写入必须幂等。
- 重复数据必须按外部 ID 去重。
