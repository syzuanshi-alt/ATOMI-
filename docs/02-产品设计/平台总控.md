# 平台总控

## 目标

平台方通过总控后台管理客户、权限、模块、授权、同步状态和交付范围。

## 一期功能

- 客户公司列表。
- 创建客户公司。
- 开通/关闭模块。
- 查看平台授权状态。
- 查看同步任务状态。
- 查看接口错误和同步日志。
- 查看成员和角色。
- 查看客户可见后台入口。

## 验收标准

- 平台方能创建至少 2 个客户公司。
- 客户之间数据隔离。
- 模块开关生效。
- 同步失败能看到错误原因。

## B-01 页面结构交付

第 1 周页面先做“能看懂业务状态”的总控，不做复杂视觉。

第一屏固定包含：

1. 今日订单。
2. 客服待处理。
3. 同步异常。
4. AI 待审核。
5. 客户运行状态。

每个数据块必须标记数据来源：

| 模块 | 数据来源 | 一期状态 |
|---|---|---|
| 今日订单 | 订单表/测试数据 | 第 3-4 周接入 |
| 客服待处理 | 客服会话/AI 建议 | 第 4-5 周接入 |
| 同步异常 | 同步运行记录 | 第 2-4 周接入 |
| AI 待审核 | AI 输出记录 | 第 5 周接入 |
| 客户运行状态 | 租户、模块、授权状态 | 第 2-3 周接入 |

当前代码骨架已在 `apps/web/src/app/page.tsx` 放入静态总控页面，用来验证页面结构和导航。
