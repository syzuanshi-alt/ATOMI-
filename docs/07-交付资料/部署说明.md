# 部署说明

## 目标

正式开发仓库部署到 GitHub 私有仓库，代码和任务通过 GitHub 管理，文档通过 Obsidian 读取 `/docs`。

## 建议部署顺序

1. 创建 GitHub 私有仓库 `ai-ecommerce-ops-platform`。
2. 推送项目初始化结构。
3. 配置 branch protection。
4. 配置 GitHub Projects。
5. 配置 Issue/PR 模板。
6. 配置环境变量和密钥。
7. 配置 Vercel 或服务器部署。
8. 配置数据库。
9. 配置飞书应用。
10. 跑内部试点数据。

## 注意

- 不把真实密钥写入 GitHub。
- 不在代码里写死客户 ID、飞书表 ID、店铺 token。
- 所有生产配置走环境变量或密钥管理。
