# B-01 绘制平台总控页面结构

## 背景

平台总控是客户和内部团队理解系统价值的入口，必须先确定信息层级。

## 负责人角色

B

## 开始日期

2026-06-01

## 截止日期

2026-06-03

## 前置依赖

阅读业务目标、客户后台、平台总控文档。

## 操作步骤

1. 设计第一屏信息：订单、GMV、客服待处理、同步异常、AI 待审核。
2. 设计客户列表和运行状态。
3. 标记每块数据来源。
4. 说明一期和二期边界。
5. 输出 Markdown 结构或原型截图。

## 验收标准

- 老板、运营、客服都能找到入口。
- 不做空洞营销页。
- 页面结构与一期范围一致。

## 测试方式

让 A/C 按页面字段检查数据来源是否存在；记录反馈。

## 是否需要 Claude 审核

否，项目负责人和 A/C 交叉审核即可。

## 是否客户可见

是，可作为演示结构。
