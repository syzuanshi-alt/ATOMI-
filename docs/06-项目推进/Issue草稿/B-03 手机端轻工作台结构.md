# B-03 手机端轻工作台结构

## 背景

手机端不应复制完整后台，而应支持老板和运营快速查看、审批、处理待办。

## 负责人角色

B

## 开始日期

2026-06-08

## 截止日期

2026-06-12

## 前置依赖

B-01、B-02 完成；A 提供权限方向；C 提供 AI/飞书状态字段。

## 操作步骤

1. 设计手机端 5 个入口：今日概览、待办、AI 日报、订单异常、客服审核。
2. 每个入口写主要字段和操作。
3. 对高风险操作设计二次确认。
4. 输出 H5/PWA 页面结构。
5. 说明哪些功能必须回电脑端处理。

## 验收标准

- 手机端能完成轻量查看和审批。
- 不出现复杂表格堆叠。
- 权限和状态反馈清楚。

## 测试方式

用手机视角走查每个页面，记录无法单手完成的操作。
