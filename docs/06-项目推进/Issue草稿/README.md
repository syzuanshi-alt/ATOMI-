# Issue草稿

这里保存第 1-2 周 GitHub 任务卡（Issues） 草稿。正式执行时，把每个草稿复制到 GitHub 任务卡（Issue），并放入 GitHub 项目看板（Project）。

## 使用规则

1. 不要把一个大任务写成一句话。
2. 每个 任务卡（Issue） 必须有负责人角色、开始日期、截止日期、操作步骤、验收标准、测试方式。
3. 真实 GitHub 账号未确定前，负责人（Owner） 先写 A/B/C。
4. 任务卡（Issue） 创建后，在本页补充 GitHub 链接。
5. 如果任务变化，先改 GitHub 任务卡（Issue），再同步回 Obsidian。
