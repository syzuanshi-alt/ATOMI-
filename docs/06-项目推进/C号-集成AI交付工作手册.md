# C号-集成AI交付工作手册

## 角色定位

C 号负责平台集成、AI 自动化与交付质量。你的任务是让外部数据能进来、AI 输出可追溯、飞书/邮件能服务业务，同时保证交付材料和测试不缺。

你每周必须优先回答三个问题：

1. 哪些平台已经具备接入条件，哪些只能先手工或半自动？
2. AI 的输出有没有保存来源、提示词（prompt）、结果、人工审核状态？
3. 本周交付是否有测试记录、风险记录、客户可见材料？

## 你负责的边界

| 负责 | 不负责 |
|---|---|
| 飞书、多维表格、抖音/TK、抖店、独立站、邮件、AI 日报、测试验收、交付文档 | 核心权限表直接变更、绕过人工审核、真实账号高风险操作 |
| 接口权限清单、字段映射、同步验证、Claude/Codex 审核材料 | 未经确认的批量邮件发送、AI 自动退款/改价/下单 |

## 第 1 周详细步骤

### C-01 配置飞书测试应用信息清单

1. 阅读 `飞书.md`、`数据同步架构.md`、`AI输出审计.md`。
2. 整理飞书接入需要的信息：
   - 飞书企业/组织信息。
   - 测试应用名称。
   - App ID / App Secret 保存方式。
   - 多维表格 app_token/table_id。
   - 机器人 webhook 是否需要。
   - 需要哪些权限。
3. 不把密钥写进 Obsidian 或 GitHub。
4. 建议创建一个 `.env.example` 字段清单，但真实值只放本地或密钥管理工具。
5. 输出飞书字段映射草案：
   - 订单异常表
   - AI 日报表
   - 达人 BD 表
   - 节点审核表

验收标准：

- 能说清楚飞书需要哪些权限。
- 能说明哪些数据可以同步，哪些不能同步。
- 没有真实密钥进入仓库。

### C-02 接口准备清单

1. 对以下平台分别整理账号、权限、接口可行性：
   - 抖音账号
   - 抖店
   - TikTok 短视频账号
   - 独立站
   - 邮件服务
2. 每个平台按以下格式写：
   - 平台名称
   - 一期目标
   - 数据字段
   - 获取方式：接口（API） / 导出 / 手工 / 半自动
   - 风险等级
   - 谁提供账号
   - 最晚准备日期
3. 将结果同步到 `接口权限表` 或 Obsidian 对应页面。

验收标准：

- 每个平台接入路径清楚。
- 不承诺暂时拿不到权限的平台。
- 第 1-2 周能用手工数据先跑起来。

## 第 2 周详细步骤

### C-03 AI 日报字段和测试清单

1. 设计 AI 日报一期字段：
   - 日期
   - 租户
   - 平台
   - 订单概览
   - 物流异常
   - 客服高频问题
   - 内容账号表现
   - 达人 BD 进展
   - 素材投流建议
   - AI 建议
   - 人工审核结论
2. 设计 AI 输出记录字段：
   - 提示词（prompt） 版本
   - 输入数据范围
   - 输出结果
   - 风险等级
   - 是否采纳
   - 审核人
3. 准备测试样例：
   - 正常销售日
   - 订单异常日
   - 客服投诉多发日
   - 物流延迟日
   - 数据缺失日

验收标准：

- AI 日报能被老板/运营看懂。
- AI 建议必须有风险等级。
- 高风险建议不能自动执行。

## AI 自动化红线

1. AI 可以分析、总结、建议、草拟回复。
2. AI 不允许直接执行退款、改价、批量发送邮件、封禁客户、删除数据。
3. AI 生成的客服回复必须支持人工确认。
4. AI 生成的邮件必须先进入待审核状态。
5. AI 日报必须保存输入数据范围和 提示词（prompt） 版本。
6. 所有 AI 结论必须允许人工标记“采纳/不采纳/待观察”。

## 邮件自动化原则

1. 一期只做模板化祝福、通知、轻营销，不做高频轰炸。
2. 必须有发送对象、发送原因、模板版本、审核人。
3. 节日祝福、物流提醒、售后关怀可作为第一批场景。
4. 涉及优惠、承诺、售后赔付的邮件必须人工审核。
5. 邮件发送失败、退订、投诉要进入日志。

## GitHub 操作步骤

1. 接口和 AI 工作先写清楚字段，再要求 A 建表、B 做页面。
2. 分支命名：`feature/C-编号-简短英文名`。
3. 每个 拉取请求（PR） 必须附测试数据或截图。
4. 接入第三方平台前，先写 `.env.example` 和权限说明。
5. 每周五准备 Claude 审核材料包。

## C号小白每日操作细化

### 每天开工

1. 打开 [[每日工作操作模板]]。
2. 打开自己的任务卡，例如 #8、#9、#10。
3. 先确认今天是整理字段、测试数据，还是写 AI 日报规则。
4. 不把真实密钥、账号密码、客户隐私写进任何文档。

### 做接口和 AI 时

1. 每个平台先写“能不能接接口（接口（API））”。
2. 不能接接口时，写“能不能导出表格或手工同步”。
3. AI 输出必须保存输入范围、提示词（提示词（Prompt））、输出结果、人工审核状态。
4. 邮件自动化必须先进入人工审核，不允许直接群发。

### 提交前

1. 更新接口权限表或对应 Obsidian 页面。
2. 把风险写清楚：账号风险、权限风险、敏感信息风险、平台规则风险。
3. 把测试样例放到任务卡或文档里。
4. 需要 Claude/Codex 辅助审核的任务，先准备输入材料。
