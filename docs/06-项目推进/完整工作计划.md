# 完整工作计划

## 计划定位

这份计划是 AI 电商自动化平台一期的总执行手册，面向 3 人小白开发团队使用。它不替代 GitHub Issues，也不替代正式 PRD；它负责把业务大图、开发顺序、每周交付、审核标准和风险控制合在一个地方。

三层执行结构固定如下：

| 层级 | 用途 | 使用方式 | 不做什么 |
|---|---|---|---|
| Obsidian | 总作战地图、长期知识库、决策沉淀 | 放完整计划、角色手册、节点审核、Claude 结论 | 不作为代码任务最终状态 |
| GitHub Issues/Projects | 每日执行主控 | 放小任务、负责人、截止日期、PR、审核状态 | 不写长篇业务解释 |
| DOCX/XLSX | 对外展示和阶段汇报 | 客户方案、PRD、推进表、责任表 | 不承担每日协作流 |

## 一期目标

一期不追求一次性接入所有平台，而是先跑通“可交付闭环”：

1. 建立多客户、多角色、多权限的平台底座。
2. 建立订单、物流、客服、内容账号、达人 BD、投流素材等核心数据模型。
3. 建立可追溯的数据同步日志，先支持手工导入和低风险 API 接入。
4. 建立电脑端后台和手机端 H5/PWA 轻工作台。
5. 建立飞书多维表格输出、AI 日报、邮件自动化、人工审核机制。
6. 用自己公司账号完成试点，再复制为客户交付模板。

## 3 人角色分工

| 角色 | 负责人称呼 | 核心责任 | 第一优先级 | 禁止事项 |
|---|---|---|---|---|
| A 号 | 平台底座与数据中台 | 项目骨架、数据库、权限、多租户、同步日志、安全边界 | 系统不乱、数据不串 | 不直接做页面美化，不随意改业务表字段 |
| B 号 | 产品后台与手机端体验 | 电脑端后台、客户后台、手机端 H5/PWA、任务中心 | 客户看得懂、团队用得起来 | 不直接改数据库迁移，不绕过权限 |
| C 号 | 平台集成、AI 自动化与交付质量 | 飞书、抖音/TK、抖店/独立站、AI 日报、邮件、测试、交付文档 | 数据进得来、AI 可追溯、交付不跑偏 | 不直接改核心权限，不让 AI 自动执行高风险动作 |

账号没有确定前，所有任务先用 A/B/C 角色推进。等 GitHub 账号确认后，再在 GitHub Project、Issue、PR、CODEOWNERS 中替换为真实账号。

## GitHub 执行规则

1. 一个 Issue 只做一件事，标题使用角色编号，例如 `A-01 创建 Next.js 项目骨架`。
2. 每个 Issue 必须包含背景、负责人、开始日期、截止日期、前置依赖、操作步骤、验收标准、测试方式、Claude 审核、客户可见性。
3. 每次代码改动必须走分支和 PR。文档小改可以直接提交，但节点计划、数据库模型、权限模型建议走 PR。
4. PR 说明必须写清楚“改了什么、怎么验证、影响哪些模块、是否需要 Claude 审核”。
5. 合并前至少由项目负责人或对应模块审核人确认。

## Obsidian 使用规则

1. Obsidian 只打开 `D:\SHKF\docs` 作为知识库根目录。
2. 每次重大决策写入 `风险与决策.md`。
3. 每周审核写入 `节点审核记录.md`。
4. Claude/Codex 审核结论不能只留在聊天窗口，必须复制到审核记录。
5. 业务范围、数据库模型、接口权限发生变化时，同步更新对应文档。

## 节点审核问题

每周五固定回答：

1. 本周是否完成计划交付？
2. 有哪些任务延期，原因是什么？
3. 是否出现业务跑偏或一期范围膨胀？
4. 是否存在权限、数据隔离、AI 自动化、邮件发送、第三方接口风险？
5. 是否有缺少测试或无法复现的结果？
6. 下周需要砍掉、提前或调整哪些任务？

## 一期不做范围

1. 不承诺全平台 API 都已接通；没有稳定授权的平台先走手工导入或半自动同步。
2. 不让 AI 自动下单、退款、改价、封号级操作。
3. 不做复杂 ERP 全模块，只做订单、物流、客户、素材、达人、日报这些闭环。
4. 不做复杂 BI 大屏堆砌，先保证核心指标真实、可追溯。
5. 不做完全无人值守客服，只做 AI 辅助回复、标签、摘要和人工确认。

## 商用交付底线

| 类别 | 底线要求 |
|---|---|
| 权限 | 不同客户数据必须隔离，管理员、运营、客服、老板角色权限清楚 |
| 数据 | 同步失败要有日志，关键字段来源可追踪 |
| AI | AI 输出必须保存 prompt、结果、审核状态，不允许黑盒决策 |
| 邮件 | 批量邮件必须有模板、退订/停止发送机制、人工审核 |
| 飞书 | 输出多维表格字段要固定，不能把敏感信息随意同步 |
| 交付 | 客户能看到方案、PRD、操作手册、验收清单、试点报告 |
