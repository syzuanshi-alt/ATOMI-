# 第1周启动执行清单

## 启动目标

第 1 周只做三件事：

1. 建立可开发的工程骨架。
2. 确认客户后台和手机端的大结构。
3. 明确飞书、抖音、抖店、独立站、TK、邮件这些外部平台的接入准备。

第 1 周不要做复杂业务代码，不要承诺全接口自动接通，不要让 AI 执行真实客户动作。

## 周一启动会

时间建议：2026-06-01 上午。

参会角色：

- 项目负责人
- A 号：平台底座与数据中台
- B 号：产品后台与手机端体验
- C 号：平台集成、AI 自动化与交付质量

会议目标：

1. 每个人打开 Obsidian，确认能看到自己的工作手册。
2. 每个人打开 GitHub，确认能看到自己的 Issue。
3. 项目负责人确认 GitHub Project 看板字段。
4. 确认第 1 周最先做的三个任务。

## A 号第 1 周执行顺序

1. 先读：
   - [[完整工作计划]]
   - [[A号-平台底座工作手册]]
   - [[03-技术架构/总体架构|总体架构]]
   - [[04-数据库模型/一期数据库模型|一期数据库模型]]
2. 打开 Issue [#1](https://github.com/syzuanshi-alt/ATOMI-/issues/1)。
3. 在 Issue 评论区写：
   - 我已开始。
   - 今天做工程目录和技术栈确认。
   - 需要确认的问题。
4. 建分支：`feature/A-01-project-skeleton`。
5. 完成骨架后发 PR。
6. PR 中必须写：
   - 目录结构。
   - 本地启动方式。
   - 没有提交真实密钥。
7. #1 完成后，开始 [#2](https://github.com/syzuanshi-alt/ATOMI-/issues/2)。

## B 号第 1 周执行顺序

1. 先读：
   - [[完整工作计划]]
   - [[B号-产品后台工作手册]]
   - [[02-产品设计/平台总控|平台总控]]
   - [[02-产品设计/客户后台|客户后台]]
2. 打开 Issue [#5](https://github.com/syzuanshi-alt/ATOMI-/issues/5)。
3. 先输出结构，不做复杂 UI。
4. 页面第一屏必须包含：
   - 今日订单/GMV
   - 客服待处理
   - 同步异常
   - AI 待审核
   - 客户运行状态
5. 输出到 Obsidian 或原型截图。
6. #5 通过后开始 [#6](https://github.com/syzuanshi-alt/ATOMI-/issues/6)。

## C 号第 1 周执行顺序

1. 先读：
   - [[完整工作计划]]
   - [[C号-集成AI交付工作手册]]
   - [[05-平台接口/飞书|飞书]]
   - [[03-技术架构/AI输出审计|AI输出审计]]
2. 打开 Issue [#8](https://github.com/syzuanshi-alt/ATOMI-/issues/8)。
3. 不写真实密钥。
4. 先整理飞书需要哪些权限、哪些表、哪些字段。

6. #8 通过后开始 [#9](https://github.com/syzuanshi-alt/ATOMI-/issues/9)。

## 每天检查

每天结束前每人回答：

1. 今天完成了什么？
2. 明天做什么？
3. 有没有阻塞？
4. 有没有需要 A/B/C 其他人配合？
5. GitHub Issue 状态是否更新？

## 周三小检查

时间：2026-06-03 下午。

检查重点：

- #1 是否有项目骨架 PR 或明确目录方案。
- #5 是否有总控页面结构。
- #8 是否有飞书权限和字段清单。
- #2、#6、#9 是否可以启动。

不通过处理：

- 如果 #1 不清楚，A 号先不进入数据库配置。
- 如果 #5 不清楚，B 号先不扩展客户后台。
- 如果 #8 不清楚，C 号先不碰真实飞书接入。

## 周五节点审核

时间：2026-06-05 下午。

必须检查：

1. A：项目骨架和数据库草案是否过关。
2. B：总控和客户后台信息架构是否过关。
3. C：飞书和接口准备清单是否过关。
4. 是否有真实密钥泄露。
5. 是否有一期范围膨胀。
6. 是否需要调整第 2 周任务。

审核结果写入：

- [[节点审核记录]]
- GitHub Project 每个 Issue 状态
- 相关 Issue 评论

## 第 1 周完成标准

- GitHub Project 看板已搭好。
- #1、#5、#8 已完成或进入待审核。
- #2、#6、#9 已启动。
- 每个 Issue 有状态、负责人、截止日期和验收证据。
- Obsidian 里能看到本周结论和风险。
