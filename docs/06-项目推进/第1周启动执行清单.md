# 第1周启动执行清单

## 第1周目标

时间：2026-06-01 至 2026-06-05。

第 1 周只做三件事：

1. A 号建立可开发的工程骨架。
2. B 号确认客户后台和手机端的大结构。
3. C 号明确飞书、抖音、抖店、独立站、TK、邮件的接入准备。

第 1 周不做复杂业务代码，不接真实客户数据，不让 AI 自动执行真实客户动作。

## 必须先读

每个人先按顺序打开：

1. [[小白协作术语表]]
2. [[每日工作操作模板]]
3. [[完整工作计划]]
4. 自己的角色手册
5. 自己的 GitHub 任务卡（Issue，任务卡）

## 周一上午启动会

时间建议：2026-06-01 09:30-10:30。

### 项目负责人要做

1. 确认每个人能打开 Obsidian。
2. 确认每个人能打开 GitHub 仓库。
3. 按 [[GitHub项目看板配置手册]] 建好项目看板。
4. 把 #1、#5、#8 状态设为“进行中”。
5. 讲清楚本周不做什么：不接真实数据、不写真实密钥、不承诺全接口自动化。

### A 号要做

1. 打开 [#1](https://github.com/syzuanshi-alt/ATOMI-/issues/1)。
2. 评论“今日开始”。
3. 读 [[A号-平台底座工作手册]]。
4. 确认项目骨架目录。
5. 不懂的技术名先查 [[小白协作术语表]]。

### B 号要做

1. 打开 [#5](https://github.com/syzuanshi-alt/ATOMI-/issues/5)。
2. 评论“今日开始”。
3. 读 [[B号-产品后台工作手册]]。
4. 先画结构，不做复杂视觉。
5. 页面第一屏必须服务业务，不做宣传页。

### C 号要做

1. 打开 [#8](https://github.com/syzuanshi-alt/ATOMI-/issues/8)。
2. 评论“今日开始”。
3. 读 [[C号-集成AI交付工作手册]]。
4. 整理飞书权限和字段。
5. 不写真实密钥。

## 周三小检查

时间：2026-06-03 下午。

必须判断：

1. #1 是否能进入待审核。
2. #5 是否能进入待审核。
3. #8 是否能进入待审核。

如果不通过：

- #1 不清楚：A 号先不做数据库。
- #5 不清楚：B 号先不扩客户后台。
- #8 不清楚：C 号先不碰真实飞书接入。

## 周五上午

每个人整理本周交付：

| 角色 | 必须准备 |
|---|---|
| A | 项目骨架、数据库草案、启动说明、密钥检查 |
| B | 总控结构、客户后台信息架构、客户是否看得懂的说明 |
| C | 飞书权限清单、接口准备清单、风险说明 |

## 周五下午节点审核

按 [[每周节点审核操作模板]] 执行。

审核顺序：

1. 先看 GitHub 项目看板。
2. 先看风险等级为“高”的任务。
3. 再看状态为“阻塞”的任务。
4. A/B/C 每人 3 分钟汇报。
5. Claude/Codex 做辅助审核。
6. 项目负责人做最终判断。
7. 结论写入 [[节点审核记录]]。

## 第1周完成标准

- GitHub 项目看板已搭好。
- #1、#5、#8 已完成或待审核。
- #2、#6、#9 已启动。
- 每张任务卡都有状态、负责人、截止日期、验收证据。
- 没有真实密钥进入 GitHub。
- Obsidian 里有本周结论和风险。
