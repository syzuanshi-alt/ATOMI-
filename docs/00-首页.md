## 工作原则

- GitHub Projects 是任务主控，Obsidian 不替代 Issue 和 PR。
- 每周节点审核必须记录到 [[06-项目推进/节点审核记录|节点审核记录]]。
- 任何影响客户数据、权限、订单、飞书、邮件、AI 自动回复的修改必须走 PR 审核。
- 飞书是业务协同出口，不是主数据库。
- AI 工具只做辅助，不直接合并代码、不直接执行高风险动作。
