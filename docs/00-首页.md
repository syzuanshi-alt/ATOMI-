## 工作原则

- GitHub 项目看板（Projects）是任务主控，Obsidian 不替代任务卡（Issue）和拉取请求（PR）。
- 每周节点审核必须记录到 [[06-项目推进/节点审核记录|节点审核记录]]。
- 任何影响客户数据、权限、订单、飞书、邮件、AI 自动回复的修改必须走拉取请求（PR）审核。
- 飞书是业务协同出口，不是主数据库。
- AI 工具只做辅助，不直接合并代码、不直接执行高风险动作。
