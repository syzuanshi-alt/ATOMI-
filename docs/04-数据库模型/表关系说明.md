# 表关系说明

## 核心关系

- 一个 `tenant` 有多个 `tenant_members`、`platform_connections`、`accounts`、`orders`、`customers`、`tasks`。
- 一个 `account` 有多个 `content_items`。
- 一个 `content_item` 有多个 `content_metrics` 快照。
- 一个 `order` 有多个 `order_items` 和多个 `shipments`。
- 一个 `customer` 可以关联多个 `orders` 和多个 `customer_threads`。
- 一个 `sync_job` 有多个 `sync_runs`。
- 一个 `ai_output` 必须归属于一个 `tenant`，并可关联日报、客服、内容、订单或任务。

## 设计注意

- 所有客户业务表必须带 `tenant_id` 或能通过父表追溯到 `tenant_id`。
- 外部平台数据必须保留 `external_id`。
- 同步写入必须避免重复。
- 删除客户数据要谨慎，优先软删除。
