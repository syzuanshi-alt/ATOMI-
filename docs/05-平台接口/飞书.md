# 飞书接口

## 一期能力

- 多维表格记录写入。
- 多维表格记录更新。
- 飞书群消息推送。
- 日报、订单异常、内容复盘、任务同步。

## 准备项

- 飞书自建应用。
- App ID。
- App Secret。
- 多维表格 token。
- 表 ID。
- 群 chat ID。
- 应用权限授权。

## 设计原则

- 飞书不是主数据库。
- 飞书写入必须幂等。
- 同步失败要写 `sync_runs` 和 `feishu_exports`。

## C-01 飞书测试应用信息清单

第 1 周只整理接入条件，不把真实密钥写入仓库。

| 项目 | 是否必需 | 填写方式 | 风险 |
|---|---|---|---|
| 飞书企业 | 是 | 项目负责人确认 | 权限不足会影响多维表格 |
| 自建应用名称 | 是 | 建议：AI电商自动化平台测试应用 | 命名不清会影响后续维护 |
| App ID | 是 | 只放本地 `.env` | 不能写进 GitHub |
| App Secret | 是 | 只放本地 `.env` | 高敏感密钥 |
| 多维表格 token | 是 | 只放本地 `.env` | 涉及业务数据 |
| 日报表 ID | 是 | 只放本地 `.env` | 字段变更要同步文档 |
| 节点审核表 ID | 建议 | 只放本地 `.env` | 便于周五审核沉淀 |
| 群机器人 webhook | 可选 | 只放本地 `.env` | 防止误发群消息 |

一期建议先建 4 张表：

1. AI 日报表。
2. 订单异常表。
3. 达人 BD 跟进表。
4. 节点审核记录表。
